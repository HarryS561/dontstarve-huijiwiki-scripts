# 饥荒维基 AI 编辑风格指南

> 本文档为 AI 执行维基编辑任务提供可操作的规范标准。基于[帮助:编写规范](https://dontstarve.huijiwiki.com/wiki/帮助:编写规范)和高质量页面提炼而成。

---

## 1. 核心原则

### 1.1 内容基准
- ✅ **以最新版本为准**：原则上以无[[模组]]的[[饥荒：联机版]]最新版本为准
- ✅ **历史版本内容**：放在"其他信息"章节下
- ❌ **不收录内容**：手游《饥困荒野》的内容暂不收录

### 1.2 语言要求
- ✅ **简体中文**：必须使用简体中文
- ✅ **官方译名**：使用官方中文译名，详见[[帮助:译名表]]
- ❌ **禁用表述**：
  - 多人版饥荒 ❌
  - 饑荒聯機版 ❌（繁体）
  - 饥荒：联机版 ✅

### 1.3 准确性原则
- 优先描述事物"具有"的特性，而非"不具有"的特性
- 避免模糊表述："一定范围""某些生物"（使用{{待补充}}模板）
- 数据来源于游戏代码解析，保持精确性

## 2. 页面结构规范

### 2.1 代码块标准顺序

```mediawiki
1. 醒目信息（{{未完成}}、{{需要帮助}}等）
2. 置顶导航（{{置顶导航}}）
3. 消歧义（{{消歧义|...}}）
4. 信息框（{{实体信息框/自动|dst|...}}）
5. 全角色台词（{{全角色台词|dst|...}}）
6. 词条摘要（第一自然段）
7. 词条正文（章节内容）
8. 脚注（{{Reflist}}）
9. 导航框（{{角色导航框}}等）
10. 分类声明（[[Category:...]]）
```

### 2.2 词条摘要格式

**标准模板**：
```mediawiki
'''中文名（{{en|英文名}}）'''是[[饥荒：联机版]]{{DST}}中的一种[类型]，[获取/生成方式]。
```

**类型选择**：
- `[[可制作]][[物品]]` - 可制作的物品
- `[[物品]]` - 不可制作的物品
- `[[可制作]][[生物]]` - 可制作的生物
- `[[生物]]` - 不可制作的生物
- `[[可制作]][[建筑]]` - 可制作的建筑
- `[[建筑]]` - 不可制作的建筑
- `实体` - 以上都不是时使用

**获取方式示例**：
```mediawiki
- 自然生成于[[洞穴世界]]的[[泥坑区群系]]
- 自然生成 1 个于[[隐士小岛]]
- 由[[蜘蛛战士]]死亡后小概率掉落 1 个
- 由[[猪人]]死亡后掉落 1 个
- 可使用[[炼金引擎]]制作
- 通过学习[[稀有蓝图]]解锁制作
```

**完整示例**：
```mediawiki
'''拆解魔杖（{{en|Deconstruction Staff}}）'''是[[饥荒：联机版]]{{DST}}中的一种[[可制作]][[物品]]。
```

### 2.3 常用章节及顺序

| 章节名称 | 适用类型 | 内容说明 |
|---------|---------|---------|
| **行为** | 生物 | 生物的行为模式 |
| **战斗** / **策略** | 生物 | 生物的对策 |
| **制作** | 材料物品 | 配合{{RRBI}}模板 |
| **皮肤** | 有皮肤实体 | 配合{{皮肤}}模板 |
| **提示** | 所有 | 使用技巧和建议 |
| **其他信息** | 所有 | 历史、原型、彩蛋、剧情、数据挖掘 |
| **bug** | 所有 | 长期未修复的Bug |
| **画廊** | 所有 | 官方宣传图等艺术作品 |
| **音频** | Boss/季节 | 战斗音乐、季节音乐 |

## 3. 用词与术语规范

### 3.1 固定术语表

| 情境 | 正确用词 ✅ | 错误用词 ❌ |
|------|-----------|-----------|
| 称呼对象 | 首次用**完整中文名**，后续用代词/简写 | 反复使用全名 |
| 称呼玩家 | **玩家**（突出操控）<br>**角色**（突出实体） | 你 ❌ |
| 角色代词 | **他/她/它**（按情况） | - |
| 其他对象 | **它** | 他/她 ❌ |
| 制作行为 | **制作** | 制造、合成 ❌ |
| 新实体出现 | **生成** | 产生、出现 ❌ |
| lootdropper | **掉落** | 生成 ❌ |
| 概率 | **概率** | 几率 ❌ |
| 生物死亡 | **死亡** | 被击败 ❌ |
| 指出凶手 | **击杀** | - |
| 距离 | **{{解释|距离单位}}** | 地皮、地皮长度 ❌ |
| 距离简称 | **单位**（使用{{解释}}后） | - |

### 3.2 数值表示规范

#### 时间
```mediawiki
✅ 1 天
✅ 4 分钟
✅ 30 秒
✅ 1.5 天
❌ 半天（应为 4 分钟）
❌ 一天半（应为 1.5 天）
```

#### 数量
```mediawiki
✅ 需要 1 个石头和 1 个树枝
✅ 掉落 3 个怪物肉
❌ 需要石头和树枝
```

#### 伤害（无模板名词）
```mediawiki
✅ 造成 20 点物理伤害
✅ 造成 10 点位面伤害
❌ 造成 20 物理伤害（缺少"点"）
```

#### 属性（有模板名词）
```mediawiki
✅ 每秒回复 30 {{生命值}}
✅ 消耗 20 {{理智值}}
❌ 每秒回复 30 点{{生命值}}（多了"点"）
```

### 3.3 空格使用规范

```mediawiki
✅ 需要 1 个 Gold Nugget
✅ 在 2021 年更新中加入
✅ 使用 Science Machine 解锁
❌ 需要1个GoldNugget（数字英文与汉字之间需空格）
```

## 4. 格式规范

### 4.1 标点符号

| 规则 | 说明 | 示例 |
|------|------|------|
| **使用全角** | 中文标点使用全角 | 。，、；：？！ |
| **句尾句号** | 每句话结尾加句号 | 这是一个物品。 |
| **专有名词例外** | 结尾有标点的专有名词除外 | 她携带了[[伯尼！]]（无需再加句号） |
| **人名间隔** | 用 · 不用 • | 奥托·冯·切斯特菲尔德先生 ✅ |
| **游戏名称** | 不使用书名号 | 饥荒：联机版 ✅<br>《饥荒：联机版》❌ |
| **版本名** | 使用双引号 | "她卖贝壳"更新 ✅ |
| **动作名** | 建议使用双引号 | 使用"检查"动作 ✅ |

### 4.2 数字格式

#### 百分比与分数
```mediawiki
✅ {{#html:MathJax3}}{{tooltip|66.7%|实为 $\frac{2}{3}$}}
✅ {{tooltip|1.5%|实为 $\frac{15}{1001}$}}
✅ 防御：50%
❌ 防御：0.5（应转换为百分比）
```

**规则**：
- 有限小数无需取近似值
- 无限小数四舍五入保留一位小数
- 概率、防御、防水等比率使用百分数（保留三位小数）

#### 区间表示
```mediawiki
✅ 3.6 ~ 4.8
❌ 3.6～4.8（全角波浪号）
❌ 3.6-4.8（减号）
```

#### 原始数值保留
```mediawiki
✅ {{#html:MathJax3}}{{tooltip|66.7%|实为 $\frac{2}{3}$}}
说明：代码中是 SHADOW_TRAP_SPEED_MULT = 2 / 3，保留原始形式并显示直观值
```

## 6. 链接与图片规范

### 6.1 链接规范

**基本规则**：
```mediawiki
✅ 同一章节内首次出现时加链接
✅ 链接指向官方中文名：[[爬行恐惧]]
❌ 不使用锚点：[[暗影生物#爬行恐惧|爬行恐惧]]
❌ 不指向错误页面：[[暗影生物|爬行恐惧]]
```

**无独立页面的实体**：
- 应建立重定向：`[[实用求雨仪式]]` → `[[书#实用求雨仪式]]`

### 6.2 图片规范

#### 小图（行内）
```mediawiki
{{Pic|32|image.png|链接}}  <!-- 32px，带链接 -->
{{Pic|64|image.png}}       <!-- 64px，不带链接 -->
```

#### 大图（正文）
```mediawiki
<!-- 右侧无信息框时 -->
[[File:图片名.png|300px|thumb|图片描述]]

<!-- 左侧或需要避免环绕时 -->
<gallery>
File:图片名.png|图片描述
</gallery>
```

**注意事项**：
- 不要只用 `right` 替代 `thumb`（手机端会环绕）
- 不建议正文插入太多大图
- 多张图片考虑使用画廊章节

#### 画廊章节
```mediawiki
==画廊==
<gallery>
She_Sell_Sea_Shell_Trailer.jpg|宣传短片截图
Update_Promo.png|更新宣传图
Concept_Art.jpg|概念图
</gallery>
```

## 8. 内容编写原则

### 8.1 信息来源优先级

1. **游戏代码解析** - 最高优先级
   - 使用项目自动解析的数据
   - 参考 `Data:DST Prefab/{code}.json`

2. **游戏内文本** - 翻译来源
   - 使用 `Module:DST Strings CN XX` 的翻译
   - 参考 `chinese_s.po` 文件

3. **既有维基内容** - 参考但需验证
   - 检查数据准确性
   - 更新过时信息

### 8.2 描述原则

#### 描述"具有"而非"不具有"

**正面例子** ✅：
```mediawiki
[[木墙]]是[[火焰燃料]]。
[[彩虹糖豆]]是[[好东西食物]]。
```

**反面例子** ❌：
```mediawiki
木墙不是[[木质食物]]。❌
木墙免疫[[冰冻]]。❌
木墙不能用于[[火源烹饪]]。❌
[[灵魂]]不是食物。❌
```

#### 特例情况（可以写"不具有"）

当对象是某大类中的特例时：
```mediawiki
✅ [[灵魂万灵药]]不受"药效强劲"技能影响。
✅ [[辣龙椒沙拉]]的[[热食]]效果不会被环境冷却。
✅ [[温蒂]]的理智值不会因为鬼魂而下降。
```

**前提**：已经介绍过该大类的共同性质。

### 8.3 物品来源原则

**规则**：在涉及"A生成/掉落B"的情况时：
- **A的页面**：详细说明何时、何种条件下生成/掉落B
- **B的页面**：简要说明来源于A，不详细展开

**示例**：

在[[猪人]]页面：
```mediawiki
==掉落==
猪人死亡后掉落 1 个{{item|猪皮}}和 1 个{{item|肉}}。
如果是满月时死亡，则掉落 2 个{{item|肉}}。
```

在[[猪皮]]页面：
```mediawiki
'''猪皮'''是...由[[猪人]]死亡后掉落。
（不详细展开满月条件等）
```

### 8.4 信息框与正文关系

**原则**：正文应尽量避免复述信息框中已有的内容

**可接受的情况**：
- 补充说明信息框无法展示的细节
- 解释特殊的解锁方式
- 说明获取方式的具体条件

**示例**：

信息框已有：
```
耐久度: 100
伤害: 68
```

正文不应写：
```mediawiki
拆解魔杖具有100点耐久度，造成68点伤害。❌
```

正文可以写：
```mediawiki
用拆解魔杖攻击不消耗耐久。✅（补充说明）
```

## 9. 特殊情况处理

### 9.1 重名实体处理

#### 修改译名（在重名情况下）

**规则**：仅在重名时允许有官中的实体使用非官中译名

**示例**：
- `bullkelp_beachedroot` → "公牛海带（海岸）"而非官中的"公牛海带"
- `carrot_planted` → "胡萝卜（自然生成）"而非"胡萝卜"
- `meatrack_hermit` → "晾肉架（隐士）"而非"晾肉架"

**操作步骤**：
1. 将新译名加入 `Data:ItemTableOverrides.tabx`（联机版）或 `Data:DSItemTableOverrides.tabx`（单机版）
2. 在信息框中使用官中覆盖回来
3. 在其余所有地方使用新译名

#### 消歧义页面

**创建消歧义页**：如"水中木（消歧义）"

```mediawiki
{{消歧义页}}
*[[水中木]]，[[饥荒：联机版]]中一个海洋[[布景]]。
*[[水中木（更新）]]，[[饥荒：联机版]]于2022年发布的内容更新。
*[[大树干]]，生成于水中木生物群系的实体。
*[[疙瘩树]]，在大树干周围生成的实体。
**[[高出平均值的树干]]，疙瘩树的特殊形态。
```

**在正文页添加消歧义提示**：
```mediawiki
{{消歧义|水中木（消歧义）}}
```

### 9.2 单机版内容处理

**规则**：联机版与单机版同名的主题，单机版内容需创建子页面

**子页面命名**：`标题/单机版`

**示例**：
- 主页面：`[[木头]]` - 联机版内容
- 子页面：`[[木头/单机版]]` - 单机版内容

**单机版DLC内容**：优先以无DLC的原版游戏为准，介绍DLC内容时需加分类并注明。

### 9.3 同义词处理

**使用重定向**：

```mediawiki
<!-- 在"怪物千层饼"页面 -->
#重定向[[怪物千层面]]
```

或：

```mediawiki
#redirect[[怪物千层面]]
```

**注意**：
- 本维基不保留英文名的重定向
- 保留主要词条，从属词条重定向

### 9.4 非实体名处理

**规则**：对于非实体名（机制名、地形名等），允许不使用官中，但需建立官中的重定向

**操作**：
1. 创建非官中名称的主页面
2. 创建官中名称的重定向页
3. 前往QQ群商讨（实际操作中）

## 10. 代码维护规范

### 10.1 适当的换行

**规则**：
- 在 wikitext 中保持适度的换行
- 换行以**一行**为宜
- **两个换行**会被识别为 `<p>`，导致另起一个自然段

**示例**：

```mediawiki
<!-- 正确：一行换行 -->
==章节1==
内容段落1。

内容段落2。

==章节2==
内容继续。

<!-- 错误：无换行（可读性差） -->
==章节1==
内容段落1。内容段落2。
==章节2==
内容继续。

<!-- 错误：过多换行 -->
==章节1==


内容段落1。


内容段落2。
```

### 10.2 复用代码原则

**规则**：如果发现写大量重复的代码，请考虑把代码做成模板。

**判断标准**：
- 相同内容在多个页面重复出现
- 重复内容不必要新建页面存放
- 可以参数化的固定格式

**示例**：
- 已有模板：`{{解释|距离单位}}`
- 可以创建新模板的情况：多页面重复相同解释文字

## 12. 高质量页面特征总结

### 12.1 寄居蟹隐士式（复杂实体典范）

**特征**：
- ✅ 多层级系统详细说明（好感度 → 任务 → 交易 → 装饰度）
- ✅ 详细数据表格（材料需求、解锁条件、得分规则）
- ✅ 完整台词系统（用折叠模板 `{{HuijiCollapsed}}` 组织）
- ✅ 清晰的机制解释（数值计算公式、触发条件）
- ✅ 分章节展示不同系统（行为、好感度、交易、装饰度）
- ✅ 使用表格展示复杂数据（任务材料表、交易解锁表）

**适用于**：NPC、复杂交互系统、多功能建筑

### 12.2 拆解魔杖式（功能性物品典范）

**特征**：
- ✅ 简洁的摘要（一句话说明核心功能）
- ✅ 公式化的规则描述（返还材料计算公式）
- ✅ 清晰的机制说明（拆解规则、特殊情况）
- ✅ 实用的提示建议（配合建造护符、刷物资技巧）
- ✅ 简明的历史记录（版本更新）

**适用于**：工具、武器、功能性物品

### 12.3 薇克巴顿式（角色页面典范）

**特征**：
- ✅ 多标签信息框（角色信息 + 实体信息）
- ✅ 背景故事完整（引人入胜的叙述）
- ✅ 特殊能力详细说明（分能力展开）
- ✅ 战术建议丰富（游戏早期、中期、后期策略）
- ✅ 皮肤套装展示（使用 `{{gallerygrid}}`）
- ✅ 其他信息丰富（游戏内容、背景故事、文化背景）

**适用于**：角色页面
